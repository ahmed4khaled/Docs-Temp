Screens / Pages

This section describes the main screens included in the current Hotel Management System admin panel. It can also be used as a guide for placing screenshots inside the final documentation website.

Login Page

The login page is the first screen for internal users. It supports authentication using either username or email, and can display demo login values for quick testing or presentation purposes.

Suggested screenshot placement: full-page login screen with logo, form, and default demo credentials.

Dashboard

The dashboard is the central page after authentication. In the current version, it acts as a simple starting point for navigating the rest of the modules.

Suggested screenshot placement: overview layout with cards, sidebar, and shortcuts.

Users & Roles

These screens manage staff accounts, role definitions, and permission assignments. They are especially important for client presentations because they show control, security, and scalability.

  • Users listing page
  • Create and edit user form
  • Roles listing page
  • Permissions assignment screen

General Settings

This page contains the configuration values that shape the hotel environment, including application identity, currency behavior, timezone, and other core settings.

Suggested screenshot placement: settings form with grouped inputs for business and regional settings.

Language Management

The language module demonstrates flexibility for multi-property or multilingual business environments. It includes language setup, translation management, and text direction support.

Room Master Data

These pages define reusable room-related information such as bed types, booking types, room types, facilities, and room sizes.

  • Bed types list and form
  • Booking types list and form
  • Room types list and form
  • Facilities page with optional image upload
  • Room sizes page

Room Management

This section covers property structure setup, including buildings and floors. It helps present the system as organized and ready for future room-level operations.

Customer Management

Customer pages manage essential guest information such as names, contact details, and basic profile data.

UI / UX Presentation Tips

  • Use consistent screenshots with the same browser size and zoom level.
  • Show real-looking sample data to make the product appear production-ready.
  • Highlight key pages first: login, dashboard, users, room master, and customer management.
  • Keep captions short and practical so clients understand the business value quickly.
