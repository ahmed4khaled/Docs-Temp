Hotel Management System

Hotel Management System is a Laravel 12 based administration platform designed to manage core hotel operations from a centralized internal dashboard. The current version is best described as a solid foundation system that already includes essential modules for users, settings, room data, and customer information, while leaving space for future hotel PMS expansion.

Features

  • Secure authentication with login using username or email.
  • User and role management with CRUD operations and dynamic permissions.
  • General hotel settings for application name, currency, timezone, and operating preferences.
  • Multi-language support with RTL and LTR layouts.
  • Room master data management for bed types, booking types, room types, facilities, and room sizes.
  • Room management for buildings and floors.
  • Customer management for storing essential guest records.
  • Clean Laravel structure ready for extension into bookings, room status, and payment tracking.

System Modules

Authentication

The system includes a custom login screen for internal staff. Users can authenticate using either username or email, and protected modules are secured through Laravel auth middleware.

Dashboard

The dashboard acts as the central entry point to the admin panel. In the current version it is intentionally simple, making it suitable for future expansion with reports, KPIs, quick actions, and hotel activity summaries.

User & Role Management

Administrators can create, update, delete, and search users and roles. Each role can store permissions in a dynamic JSON structure, allowing flexible access control across different modules.

  • Role CRUD with search and pagination.
  • User CRUD with active and inactive status.
  • Permission levels such as View, Create, Edit, Delete, and Full.
  • Role permissions can be copied directly to users during create or update actions.

General Settings

This module controls the main configuration of the hotel environment, including business identity and operational defaults.

  • Application name and hotel title.
  • Fiscal starting month and initial business date.
  • Currency, symbol position, decimal settings, and timezone.
  • Logo dimensions and core business preferences.

Language Management

The platform supports multilingual administration through a dedicated language management module. Languages can be enabled, disabled, set as default, and edited from the admin panel.

  • Multi-language support with translation key management.
  • RTL and LTR interface direction support.
  • Flexible translation file handling for system messages.

Room Master Data

This module stores reusable room-related definitions used across the hotel setup.

  • Bed types
  • Booking types
  • Room types
  • Facilities with optional image support
  • Room sizes

Room Management

Room management currently focuses on organizing the property structure itself.

  • Buildings management
  • Floors management
  • Search by name or code with validation rules for data consistency

Customer Management

The customer module stores core guest information and supports basic CRUD operations for front office or admin-level customer records.

Installation Guide

  1. Clone the project into your local environment.
  2. Run composer install.
  3. Run npm install.
  4. Copy .env.example to .env.
  5. Generate the application key using php artisan key:generate.
  6. Configure database credentials in the .env file.
  7. Either run php artisan migrate or import the provided hotel.sql file.
  8. Start the application using php artisan serve.
  9. Start frontend assets using npm run dev or use composer run dev.

Folder Structure

  • app/Http/Controllers contains controller logic for all modules.
  • app/Models contains Eloquent models for hotel entities.
  • routes/*.php organizes routes by module.
  • resources/views contains Blade templates and interface files.
  • database/migrations contains database migration files.
  • hotel.sql can be used for direct database import if preferred.
